# MOA Bodychart PWA

MOA BUSAN의 통증 바디차트 · 자세·정렬 관찰 웹앱입니다.

## 포함 기능

- 아이폰·안드로이드·데스크탑 반응형 UI
- 홈 화면 설치(PWA)
- 오프라인 실행
- 결과 이미지 미리보기
- 모바일 네이티브 사진 저장·공유
- PNG 파일 다운로드
- 현재 평가 자동저장·복구
- JSON 백업·복원
- 자동 업데이트 확인 및 업데이트 배너

## GitHub Pages에 올리는 방법

1. 새 GitHub 저장소를 만듭니다. 예: `moa-bodychart`
2. 이 폴더 안의 파일과 `icons` 폴더를 **그대로 저장소 루트**에 업로드합니다.
3. GitHub 저장소에서 `Settings` → `Pages`로 이동합니다.
4. `Build and deployment`에서 `Deploy from a branch`를 선택합니다.
5. Branch는 `main`, 폴더는 `/(root)`를 선택하고 저장합니다.
6. 배포 주소가 생성되면 아이폰 Safari 또는 안드로이드 Chrome에서 접속합니다.

중요: `index.html`, `manifest.webmanifest`, `sw.js`, `version.json`, `icons/`의 상대 위치를 바꾸지 마세요.

## 모바일에서 사진 저장

상단의 `사진 저장` 버튼을 누릅니다.

1. 현재 탭 또는 통합 리포트를 선택합니다.
2. 미리보기에서 `사진 저장·공유`를 누릅니다.
3. 아이폰은 공유창에서 `이미지 저장`을 선택하면 사진 앱에 저장할 수 있습니다.
4. 공유 기능을 사용할 수 없는 브라우저에서는 `PNG 다운로드`를 사용합니다.

## 새 버전 배포 방법

업데이트할 때는 다음 두 곳의 버전을 동일하게 올립니다.

- `index.html`의 `APP_VERSION`
- `version.json`의 `version`

예: `1.0.0` → `1.0.1`

그 뒤 변경 파일을 GitHub에 업로드하면 앱이 `version.json`을 자동 확인하여 사용자에게 업데이트 배너를 보여줍니다.

`sw.js` 자체를 수정한 경우에도 브라우저가 새 서비스 워커를 감지하여 업데이트 배너를 표시합니다.

## 데이터 주의

입력한 내용은 서버로 전송하지 않고 현재 기기의 브라우저 저장소에 보관됩니다. 앱 삭제, 브라우저 데이터 삭제, 기기 변경 시 데이터가 사라질 수 있으므로 중요한 평가 전후에는 `더보기` → `데이터 백업`을 사용하세요.

공용 배포판에서는 환자 실명·주민번호·연락처 등 민감정보 입력을 피하고 환자 이름/코드 사용을 권장합니다.
